TCP 代理¶

  • TCP 代理 架构概览

  • v3 API 参考

  • 过滤器的名称应该配置为 envoy.filters.network.tcp_proxy 。

动态集群选择¶

TCP 代理过滤器使用的上游集群可以由其他网络过滤器根据每个连接动态设置，方法是在 envoy.tcp_proxy.cluster 字段下设置每个连接的状态对象。见实现详情。

路由到主机子集¶

可以配置 TCP 代理，让它路由到上游集群内部的主机子集。

为了定义合适的上游主机必须匹配的元数据，请使用以下字段之一：

  1. 使用 TcpProxy.metadata_match 为单个上游集群定义所需的元数据。

  2. 使用 ClusterWeight.metadata_match 为权重集群定义所需的元数据。

  3. 使用 TcpProxy.metadata_match 和 ClusterWeight.metadata_match 的组合为权重上游集群定义所需的元数据（后者的元数据将在前者的基础上合并）。

此外，元数据可以由 StreamInfo 上的早期网络过滤器设置。设置元数据必须在 TcpProxy 过滤器调用 onNewConnection() 前发生，以此来影响负载均衡。

统计信息¶

TCP 代理过滤器即会发出自己的下游统计信息，也会在适用的情况下发出许多 上游集群统计信息 。下游统计信息以 tcp.<stat_prefix>. 为根，如下所示的统计信息：

类型

downstream_cx_tx_bytes_total

Counter

downstream_cx_tx_bytes_buffered

Gauge

downstream_cx_rx_bytes_buffered

Gauge

Counter
